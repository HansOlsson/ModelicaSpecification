# Standard type coercion when `spatialDistribution` is used with `Integer` arguments #3795

## Main change

Add support for discontinuity propagation and other types than `Real` in `spatialDistribution`, https://github.com/modelica/ModelicaSpecification/pull/3795

## Reason

Port recent generalizations for `delay` to `spatialDistribution`, making it an event-generating function which is able to propagate discontinuities and support transportation of discrete-valued types (`Boolean`, `Integer`, and enumeration types).

## Breaks

``` modelica
  Real x;
  Real out1;
equation
  (out1, ) = spatialDistribution(0, 1, x, true, {0, 1}, {2, 3});
```

Previously, when `spatialDistribution` only supported transportation of `Real` signals, all of the literals were cocerced to `Real`, and it was not a problem that the initial buffer state was a continuous transition from 2.0 to 3.0.
With the new semantics, this `spatialDistribution` call models transport of `Integer` signals, and then it is not allowed for the initial buffer state to have continuous transitions between values.

The remedy to get back the old behavior is to make sure at least one of the input expressions with signal values is of `Real` type.
Here, all relevant literals are made `Real` for clarity:
```
  (out1, ) = spatialDistribution(0.0, 1.0, x, true, {0, 1}, {2.0, 3.0});
```
